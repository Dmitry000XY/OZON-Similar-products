# Архив

Эта папка хранит старые EDA-материалы и код, который больше не является частью
production-пайплайна.

## Почему архивный код лежит в `docs/archive/code/`

- Это часть документации и исторического контекста, а не production-слой.
- Размещение в `src/` создало бы ложный сигнал, что модуль поддерживается как runtime-код.
- Размещение в `notebooks/` смешало бы исполняемые отчёты с историческими артефактами.
- Отдельная top-level папка `archive/` размывает ownership: здесь архив логично живёт
  рядом с архитектурными пояснениями в `docs/`.

## Исполняемость и статус

- Архивный код/примеры в `docs/archive/code/` считаются historical reference.
- Они не входят в production API и не являются обязательной частью исполняемого контура.
- Такие файлы не должны импортироваться из `src/`, `tests/`, `scripts`.

## Можно ли импортировать архивный код

Нельзя. Импорт из `docs/archive/code/` в runtime-слои запрещён guardrail-проверками.

## Что делать, если нужно переиспользовать часть архивного кода

1. Перенести нужный фрагмент в соответствующий production-layer в `src/`.
2. Адаптировать под текущие контракты схем и конфигов.
3. Добавить/обновить тесты и документацию.
4. После миграции оставить в архиве только ссылку на новое production-место.
