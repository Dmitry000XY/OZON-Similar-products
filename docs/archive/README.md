# Архив

Эта папка хранит старые исследовательские материалы и код, который больше не участвует в рабочем запуске проекта.

Архив нужен как исторический контекст: по нему можно понять, какие подходы уже пробовались и почему часть кода не вошла
в текущую структуру проекта.

## Почему архив лежит в `docs/archive/`

Архивные материалы лежат рядом с документацией, потому что они не являются рабочим слоем проекта.

Если оставить такой код в `src/`, это будет выглядеть так, будто модуль поддерживается и входит в основной пакет.

Если перенести его в `notebooks/`, архив смешается с актуальными исследовательскими ноутбуками и проверочными отчётами.

Отдельная папка `docs/archive/` показывает правильный статус этих материалов:

```text
это полезно для истории проекта
это не часть рабочего запуска
```

## Статус архивного кода

Код и примеры в `docs/archive/code/`:

* не входят в основной пакет;
* не являются публичным API проекта;
* не используются в конвейере обработки;
* не должны импортироваться из `src/`, `tests/` или `scripts/`.

Архивный код может быть полезен как пример старого подхода, но на него не нужно опираться при разработке новой логики.

## Можно ли импортировать код из архива

Нет.

Импорт из `docs/archive/code/` в рабочие слои запрещён guardrail-проверками.

Рабочий код должен жить в:

```text
src/ozon_similar_products/
```

Тесты должны проверять код из `src/`, а не код из архива.

## Что делать, если архивный код снова понадобился

Если из архива нужен фрагмент логики, его нужно не импортировать напрямую, а перенести в нормальное место внутри
`src/ozon_similar_products/`.

Порядок такой:

1. Найти подходящий рабочий модуль.
2. Перенести туда нужную логику.
3. Адаптировать код под текущие контракты таблиц и конфигов.
4. Добавить или обновить тесты.
5. Обновить документацию рядом с модулем.
6. В архиве оставить короткую пометку, куда перенесена актуальная версия.

## Коротко

`docs/archive/` — это место для старых материалов, а не запасной рабочий модуль.

Архив можно читать, но нельзя использовать как зависимость текущего проекта.

Если старая логика снова нужна, её нужно перенести в `src/`, привести к текущим контрактам и покрыть тестами.
