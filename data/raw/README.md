# Исходные данные

Эта папка содержит локальные входные данные проекта **OZON Similar Products**.

Данные из этой папки не должны коммититься в Git. В репозитории хранится только этот README и служебная структура папок.

## Структура папки

Ожидаемая структура:

```
data/raw/
  archives/
    product_information.tar.gz
    user_actions.tar.gz
  product_information/
  user_actions/
```

Где:

| Путь                   | Что лежит                               |
|------------------------|-----------------------------------------|
| `archives/`            | исходные архивы данных                  |
| `product_information/` | распакованный справочник товаров        |
| `user_actions/`        | распакованные пользовательские действия |

Важно: `archives/` здесь — это папка с исходными архивами данных. Она не связана с удалёнными историческими материалами
документации.

## Ожидаемые архивы

Для подготовки данных нужны два архива:

* `product_information.tar.gz`;
* `user_actions.tar.gz`.

Они должны лежать в папке:

```
data/raw/archives/
```

## Подготовка данных

Из корня проекта выполните:

```
uv run python scripts/prepare_raw_data.py
```

После успешного запуска должны появиться подготовленные папки:

```
data/raw/product_information/
data/raw/user_actions/
```

Проверить структуру проекта и наличие данных можно командой:

```
uv run python scripts/check_project_structure.py
```

## Что не нужно делать вручную

Не стоит вручную менять распакованные данные, если они уже подготовлены скриптом.

Если нужно пересобрать данные из архивов, лучше использовать:

```
uv run python scripts/prepare_raw_data.py --force
```

Так проще сохранить воспроизводимость локального запуска.

## Коротко

Положите архивы в `data/raw/archives/`, затем запустите:

```
uv run python scripts/prepare_raw_data.py
```

После этого рабочие данные будут подготовлены в `data/raw/product_information/` и `data/raw/user_actions/`.
