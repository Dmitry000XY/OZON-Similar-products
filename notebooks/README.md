# Исследовательские ноутбуки

В этой папке мы храним ноутбуки, которые использовались на исследовательском этапе проекта **OZON Similar Products**.

Через ноутбуки мы разбирались с данными, проверяли первые гипотезы и смотрели, насколько пользовательские события
подходят для построения блока «Похожие товары». Основная рабочая логика после этого была вынесена в Python-модули
внутри [`../src/ozon_similar_products/`](../src/ozon_similar_products/).

## Что здесь есть

```text
notebooks/
  01_eda.ipynb
```

| Ноутбук                        | Что в нём разбирается                                                                            |
|--------------------------------|--------------------------------------------------------------------------------------------------|
| [`01_eda.ipynb`](01_eda.ipynb) | первичный анализ пользовательских событий, товаров и готовности данных к построению рекомендаций |

## Зачем мы использовали ноутбуки

Перед тем как писать основной конвейер обработки, нам нужно было понять данные:

* какие типы действий есть в логах;
* насколько часто пользователи взаимодействуют с товарами;
* какие действия можно считать товарными сигналами;
* как данные распределены по датам;
* есть ли пропуски или странные значения;
* можно ли строить сессии по этим событиям;
* достаточно ли совместных действий пользователей для первого варианта похожих товаров.

Ноутбук [`01_eda.ipynb`](01_eda.ipynb) фиксирует этот первый этап: не финальную реализацию, а исследование, из которого
выросла текущая структура проекта.

## Как ноутбуки связаны с основным кодом

Ноутбуки в этом проекте — это не отдельный способ запуска рекомендаций.

Рабочий запуск находится в скриптах и модулях:

```text
scripts/
src/ozon_similar_products/
```

Подробнее:

* [`../scripts/README.md`](../scripts/README.md) — пользовательские команды запуска;
* [`../src/ozon_similar_products/`](../src/ozon_similar_products/) — основной Python-пакет;
* [`../docs/architecture.md`](../docs/architecture.md) — архитектура проекта.

Ноутбук помогает понять, почему проект устроен именно так, но не заменяет основной конвейер обработки.

Например:

| Исследование в ноутбуке          | Рабочая реализация                                                                                                                      |
|----------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------|
| анализ исходных событий          | [`data`](../src/ozon_similar_products/data/README.md)                                                                                   |
| проверка товарных действий       | [`preprocessing`](../src/ozon_similar_products/preprocessing/README.md)                                                                 |
| анализ сессий                    | [`preprocessing`](../src/ozon_similar_products/preprocessing/README.md)                                                                 |
| анализ популярности товаров      | [`features`](../src/ozon_similar_products/features/README.md)                                                                           |
| первые идеи по похожести товаров | [`retrieval`](../src/ozon_similar_products/retrieval/README.md)                                                                         |
| проверки качества и диагностика  | [`diagnostics`](../src/ozon_similar_products/diagnostics/README.md) и [`evaluation`](../src/ozon_similar_products/evaluation/README.md) |

Если какая-то проверка из ноутбука стала нужна регулярно, её лучше переносить в код проекта, чтобы её можно было
запускать повторяемо и покрывать тестами.

## Как запустить ноутбук

Сначала установите зависимости:

```bash
uv sync
```

Затем подготовьте локальные данные:

```bash
uv run python scripts/prepare_raw_data.py
```

Проверить, что структура проекта и данные на месте:

```bash
uv run python scripts/check_project_structure.py
```

После этого можно открыть ноутбук:

```text
notebooks/01_eda.ipynb
```

Ноутбук ожидает, что данные лежат в стандартных папках проекта:

```text
data/raw/
data/processed/
outputs/
```

Подготовка данных и команды запуска описаны в [`../scripts/README.md`](../scripts/README.md).

## Что важно при работе с ноутбуками

Мы стараемся держать ноутбуки как понятные исследовательские отчёты, а не как набор случайных ячеек.

В хорошем ноутбуке должно быть понятно:

* какую задачу мы проверяли;
* какие данные использовали;
* какие выводы получили;
* что из этого перешло в основной код;
* какие вопросы остались открытыми.

Не стоит хранить в ноутбуках личные пути, токены, большие выгрузки данных или логи, которые невозможно воспроизвести.

## Когда нужен новый ноутбук

Новый ноутбук имеет смысл добавлять, если появляется отдельная исследовательская задача.

Например:

```text
02_sessions_analysis.ipynb
03_item_pairs_diagnostics.ipynb
04_fallback_analysis.ipynb
05_recommendation_quality.ipynb
```

Если задача уже стала частью регулярного запуска, лучше делать не ноутбук, а отдельную функцию, модуль или скрипт.
